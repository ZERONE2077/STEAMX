# 基于 OpenSteamTools 内核的游戏入库脚本规划

> 状态：待补充  
> 创建日期：2026-07-03  
> 目标：规划一个脚本工具，主要完成两件事：把 OpenSteamTools 部署到本机 Steam 环境内；从清单库下载游戏清单到 Steam 的 Lua 文件夹，并提供必要的备份、校验和日志。

## 1. 项目目标

### 1.1 核心任务

脚本只围绕两个主流程设计：

1. 部署 OpenSteamTools 到 Steam 内。
2. 从清单库下载清单到 Steam 的 Lua 文件夹。

最小可用版本不先做复杂 UI、不做账号相关功能、不做完整游戏管理器。

### 1.2 要解决的问题

- 自动识别本机 Steam 安装路径。
- 确认 OpenSteamTools 需要部署到 Steam 内的哪些目标目录。
- 下载或更新 OpenSteamTools 部署文件。
- 找到 Steam 的 Lua 文件夹。
- 从指定清单库下载 Lua 清单。
- 支持按 appid、游戏名或清单链接下载。
- 操作前备份将被覆盖的文件。
- 输出清晰日志，便于确认成功或定位失败。

### 1.3 非目标

- 不开发 Steam 客户端替代品。
- 不处理账号凭据、支付、库存、交易等敏感功能。
- 不绕过平台安全机制或用户授权边界。
- 不修改与入库无关的 Steam 文件。
- 不在第一版实现完整 GUI。

## 4. 功能规划

### 4.1 第一阶段：最小可用版本

- 识别 Steam 安装路径。
- 识别或手动指定 Steam Lua 文件夹。
- 从本地路径部署 OpenSteamTools 到 Steam 内。
- 从固定清单库下载单个 Lua 清单。
- 将 Lua 清单保存到 Steam Lua 文件夹。
- 覆盖前自动备份旧文件。
- 输出执行日志。
- 支持失败后恢复备份。

### 4.2 第二阶段：远程更新与批量

- 从 GitHub Release 或指定 URL 下载 OpenSteamTools 文件。
- 支持批量下载多个 appid 的 Lua 清单。
- 支持本地清单缓存。
- 支持下载失败重试。
- 支持代理配置。
- 支持清单版本比对或文件哈希校验。

### 4.3 第三阶段：体验与自动化

- 自动检测 Steam 是否运行。
- 可选自动关闭和重启 Steam。
- 清单源管理。
- 执行前预检。
- 执行后验证目标文件是否存在。
- 可选增加简单 GUI 或交互式命令行。

## 5. 技术设计要点

### 5.1 Steam 路径与 Lua 目录识别

优先级建议：

1. Windows 注册表读取 Steam 安装目录。
2. 根据 Steam 根目录推导 Lua 文件夹。
3. 扫描常见 Steam 路径。
4. 用户手动指定 Steam 路径或 Lua 文件夹。

需要处理：

- 路径不存在。
- Steam 安装在非系统盘。
- 用户权限不足。
- 文件被 Steam 占用。
- Lua 文件夹不存在时是否自动创建。

### 5.2 OpenSteamTools 部署流程

需要确认：

- OpenSteamTools 的来源：本地文件、GitHub Release、源码编译产物。
- 哪些文件需要复制到 Steam 内。
- 目标目录是否固定。
- 是否需要删除旧版本文件。
- 是否需要关闭 Steam。
- 是否需要管理员权限。

建议流程：

1. 检测 Steam 是否存在。
2. 检测 Steam 是否正在运行。
3. 检查 OpenSteamTools 源文件是否完整。
4. 生成部署计划，列出复制目标。
5. 备份目标目录中的旧文件。
6. 复制 OpenSteamTools 文件。
7. 校验文件数量、大小或哈希。
8. 写入日志。

### 5.3 清单下载到 Lua 文件夹

需要确认：

- 清单库基础地址。
- 单个 appid 对应的下载 URL 规则。
- 清单是单个 `.lua` 文件，还是压缩包。
- 下载后文件名规则。
- 目标 Lua 文件夹路径。
- 是否允许覆盖同名 Lua 文件。
- 是否需要校验清单完整性或 appid 匹配。

安全建议：

- 如果 Lua 来自远程源，默认不要直接无沙箱执行。
- 下载阶段只把 Lua 当作文件保存，不在脚本内执行。
- 如需读取内容，只做静态校验，例如 appid、文件头、基本语法特征。

### 5.4 文件写入与回滚

需要明确：

- 目标文件路径。
- 是否覆盖已有文件。
- 覆盖前备份策略。
- 失败时回滚策略。
- Steam 正在运行时是否允许操作。

建议流程：

1. 预检路径和权限。
2. 生成部署计划。
3. 展示将要改动的文件。
4. 创建备份。
5. 执行写入。
6. 校验结果。
7. 记录日志。

## 7. 命令行草案

```powershell
# 检测环境
steamx check

# 部署 OpenSteamTools 到 Steam
steamx deploy-ost

# 下载单个 Lua 清单到 Steam Lua 文件夹
steamx lua install --appid 123456

# 从指定 URL 下载 Lua 清单
steamx lua install --url https://example.com/123456.lua

# 批量下载 Lua 清单
steamx lua install --list appids.txt

# 回滚最近一次操作
steamx rollback --last
```

## 8. 日志与备份

### 8.1 日志内容

- 运行时间。
- Steam 路径。
- Steam Lua 路径。
- 清单来源。
- appid。
- OpenSteamTools 来源和目标路径。
- 下载结果。
- 文件写入结果。
- 备份路径。
- 错误堆栈。

### 8.2 备份策略

- 每次执行生成独立备份目录。
- 备份目录名称包含时间戳。
- 保存部署计划和原始文件。
- 回滚时严格只恢复本次改动文件。

## 9. 风险与边界

### 9.1 技术风险

- OpenSteamTools 接口不稳定。
- OpenSteamTools 部署文件结构变化。
- Steam 客户端更新导致 Lua 目录或加载方式变化。
- 远程清单格式不统一。
- Lua 执行存在安全风险。
- 文件被占用导致写入失败。

### 9.2 合规与安全边界

- 不采集 Steam 账号密码、令牌、Cookie。
- 不上传用户本地 Steam 配置。
- 不绕过用户授权。
- 对远程脚本默认不授予系统权限。
- 发布前明确依赖项目 License。
